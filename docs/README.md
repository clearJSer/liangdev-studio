# Docs

本文档的职责：说明 `docs/` 目录的用途和文档归档规则。

`docs/` 用于存放长期维护的工程文档，例如开发流程、架构说明、协作指南和工程实践。

## 目录约定

- `development/`：开发流程、AI 协作方式、提交与验收规范。
- `architecture/`：系统架构、模块边界、技术选型记录。需要时再创建。

## 当前文档

- [Codex 协作开发指南](development/codex-collaboration.md)

## 不放在这里的内容

- 产品需求、页面规划、业务规则和设计规格放在 `specs/`。
- agent 自动读取的规则放在根目录 `AGENTS.md`。
- 项目快速介绍放在根目录 `README.md`。
