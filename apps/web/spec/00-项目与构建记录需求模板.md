# 内容命名规则

- 项目 slug 使用小写英文 kebab-case，例如：`yiforge-studio-website`。
- 文章 Markdown 文件名使用小写英文 kebab-case，并作为文章 URL slug，例如：`building-studio-website-with-ai.md`。
- 中文标题写在 Markdown frontmatter 的 `title` 字段中，不直接作为文件名。
- slug 优先保证简短、可读、稳定，不追求完整翻译中文标题。
- 不建议使用随机字符串或无意义序号作为 slug。

# 静态部署约束

- 不使用数据库。
- 不使用 CMS。
- 不使用服务端运行时读取文章。
- 不依赖动态 API 加载内容。
- 所有项目详情页和文章详情页都在构建阶段生成。
- 新增文章后，通过重新构建和部署生成新的静态页面。
